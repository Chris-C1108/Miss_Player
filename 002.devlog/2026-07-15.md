## 标签栏交互触控重构与关联发光系统优化

## 1. 核心需求与实现功能

### 1.1 触控面积扩充与边缘发光无损逃逸 (Layout & Touch Size)
- **标签胶囊高度升至 38px**：为了彻底解决原有标签页过矮导致的移动端误触问题，取消了胶囊上下的 padding，并将 `.tm-tab-pill` 物理高度固定拉高至 **`38px`**，触控面积显著增加。
- **清单按钮同步放大**：右侧折叠面板的清单按钮同步变大为 `38px × 38px` 的圆形，与标签栏在视觉基准线上完美平齐对齐。
- **发光溢出逃逸技巧**：
  - 由于水平滚动容器 `.tm-tab-scroll-container` 必须使用 `overflow-x: auto; overflow-y: hidden;`，这会导致子元素的高亮外阴影（`box-shadow`）被浏览器强行裁剪截断。
  - 通过在容器上配置 `padding: 6px 0;`（增加物理可视高度至 `50px` 提供发光缓冲区）与 **`margin: -6px 0;`**（利用负外边距对冲，使 DOM 布局占用空间保持为原样 `38px`）的组合拳，成功实现了发光效果在不撑开上下元素的前提下，向上和向下“溢出”重叠渲染，解决了发光截断问题。

### 1.2 新建空标签页与交互对齐规则重定义 (UI & Interaction Flow)
- **取消文字“+新标签页”，改为灰色默认胶囊**：初始化时在末尾显示一个未激活的中性灰空标签页 `[ A --:--:-- ] [ B --:--:-- ]`，激活后才会按照调色板顺序动态分配高亮颜色。
- **取消强制左侧对齐**：激活新的空标签页或对其进行编辑时，不再强制让当前标签靠左对齐。用户在任何状态下都可以自由触摸滑动（或通过鼠标滚轮）横向滚动标签页栏。
- **点击闪烁节点**：点击已保存的标签页胶囊时，不仅会跳转或循环，还会如同点击评论区时间戳一样，在进度条的对应起始节点上进行醒目的黄色高亮闪烁指示。

### 1.3 被动时间关联发光与循环呼吸灯特效 (Visual Effects)
- **时间线常驻关联发光 (`.near-playback`)**：
  - **单时间戳**：当前播放时间在标记点 **`±5秒`** 范围内时，胶囊背景和边框柔和发光。
  - **时间区间**：当前播放时间处于区间内，或在起点前 **`5秒`** 至终点后 **`5秒`** 的过渡区间内时，胶囊发光点亮。
- **循环播放环绕呼吸灯 (`.looping`)**：
  - 当某个区间被激活并处于循环播放状态时，标签胶囊会绑定一个周期为 2s 的 CSS 关键帧动画 `tm-glow-breathing`。光环的羽化阴影半径与边框透明度会随时间呈现强弱明暗交替的呼吸灯特效。

### 1.4 编辑态阻断循环防止干扰打点 (Logic Cut-Off)
- 进入已有标签编辑模式（长按）或开始在空白灰色占位符上点击录入 `A` 点时，程序会自动调用 `disableLoop()` 重置激活状态并关闭当前的区间循环。
- 这解决了用户在进度条上寻求打点时间（Seek）时，由于循环引擎不断强制把播放头 Snap 回区间起点的冲突，让用户能心无旁骛地进行 A、B 时间的精准录入。

### 1.5 滚动溢出边界修复 (Scroll Padding)
- 在水平滚动容器末尾增加了 **`padding-right: 44px;`** 的占位裕量。
- 使得最后一个新建标签页在向左滑动时，能完全越过右侧 `36px` 宽度的渐变淡化遮罩（`mask-image`）以及右侧清单按钮，以 100% 的不透明度完整地展现在用户视野内，方便点击保存 `✓` 或取消 `✕`。

### 1.6 Safari 兼容性 Bug 修复 (Browser Compatibility)
- 修复了 Safari 浏览器下进度条上正在播放的活动高亮色块（`.tm-active-loop-played`）偶尔无法显示的兼容性 Bug。

## 2. 踩坑教训与开发经验

### 2.1 CSS 容器 `overflow` 对 `box-shadow` 裁剪的避坑与双重穿透方案 (布局与渲染经验)
- **教训**：
  1. 当一个容器设定为 `overflow-x: auto; overflow-y: hidden` 时，子元素的 `box-shadow` 必定会被裁断。不仅垂直方向如此，**在水平滚动起点处的首个元素，其左侧的外发光也会因为碰到滚动容器的左侧边界而被强行切断**。
  2. 更隐蔽的是，当父容器（如 `.tm-control-buttons`）同时配置了 **`border-radius`** 和 **`backdrop-filter`** 时，浏览器为了计算背景模糊，会自动为该容器开启独立的**合成渲染层（Compositing Layer）并强制应用剪裁**。即便容器的 `overflow` 设为 `visible`，任何溢出容器边界的子元素阴影也会被无情剪切。
- **经验**：
  1. **父容器视觉脱离（伪元素化）**：为了解除合成层剪裁限制，将 `.tm-control-buttons` 自身的 `background-color`、`backdrop-filter`、`border` 和 `box-shadow` 全部剥离，移入其新建的 **`::before` 伪元素** 中。
  2. **圆角动态继承**：伪元素使用 **`border-radius: inherit;`**，完美动态继承父容器在响应式媒体查询中的所有圆角变动（如全屏时圆角归零），且绝不影响子元素的溢出渲染。
  3. **横向滚动空间与起点对齐双重扩容**：
     - 将滚动容器的 padding 扩大到 **`padding: 16px 44px 16px 16px;`**，对冲外边距设为 **`margin: -16px 0 -16px -16px;`**。
     - **原理**：通过在左侧同样拓展 `16px` 的内边距并以负外边距抵消，使滚动容器的实际可视绘制边界整体向左位移 `16px`。这既为首个标签的左侧发光预留了绝对充足的独立绘制区域，又通过 padding-left 将标签页物理位置推回原位，保持了与页面左边缘的严谨视觉对齐，彻底解决了左侧阴影剪裁的问题。
  4. **指针事件双重穿透**：由于扩容后滚动容器会物理遮挡上方的进度条，给滚动容器配置 **`pointer-events: none;`**，并对标签胶囊配置 **`pointer-events: auto;`**。保证用户操作穿透空区域，不产生任何触控拦截。

### 2.2 播放器 `timeupdate` 常驻绑定与动态解绑陷阱 (事件经验)
- **现象**：新开发的被动时间关联发光功能在视频播放时毫无反应。
- **原因**：之前的循环播放引擎为了性能优化，仅在“开启循环”时向视频挂载 `timeupdate` 监听器，并在“取消循环”时立刻解绑。这意味着在普通播放时，`timeupdate` 的回调函数根本就未在视频上注册，被动发光状态更新自然失效。
- **经验**：对于需要常驻监测视频播放进度的业务逻辑（如时间线发光、弹幕检测、被动标记等），应当在 `LoopManager` 组件初始化时向 `targetVideo` 绑定一个生命周期常驻的 `timeupdate` 监听器，而只将专门用于截断重设播放位置的 A-B 边界循环检查限制在 `loopActive` 状态下执行。

### 2.3 触控尺寸（Touch Target Size）的无缝适配 (设计经验)
- 移动端 Web 设计中，过于细长或者偏矮的交互组件在实际屏幕操作中体验极差。原先通过 vertical padding 控制胶囊高度的做法是不稳定的，直接通过 `height: 38px; box-sizing: border-box;` 可以提供一个极为扎实的触控热区，让用户即使处于单手握持状态，也能非常精准、舒适地横向轮转、点击以及长按标签页。
