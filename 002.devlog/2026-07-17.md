# 开发日记 — 2026年7月17日
## 控制面板无边框扁平化、动态环境色毛玻璃适配与标签页胶囊内进度条系统

---

## 1. 核心需求与实现功能

### 1.1 控制面板无边框净化与防塌陷设计
* **彻底清除硬边框与高反光**：响应“去除无聊特效”的要求，移除了之前为模拟 Apple 液体玻璃效果所创建的 `.tm-control-buttons::before` 伪元素。将背景、模糊和阴影重新收纳回 `.tm-control-buttons` 主类中，彻底删除了顶部高光反光（`0.16` 透明度）和四周实体硬描边。
* **显微级“防塌陷”内阴影 (Option A)**：为了在无边框状态下防止控制面板与纯黑视频背景发生视觉融合（塌陷），引入了一层极其低调的 `5%` 透明度白色内描边（`inset 0 0 0 1px rgba(255, 255, 255, 0.05)`），既保障了边界的可读性，又使上边缘不再出现突兀的偏亮条。

### 1.2 动态环境色毛玻璃适配 (Ambient Tinting Glass)
* **色彩呼吸机制**：原本的控制面板使用高不透明度中性灰（`rgba(20,20,20, 0.82)`），覆盖在带有宿主暖紫红色调（Wine Red）的评论区上方时显得死板脏乱。
* **低饱和透光融合**：将面板不透明度降低至 `52%` (`rgba(10, 10, 10, 0.52)`)，并配合高斯模糊半径升级至 `24px`。使得底部的暗紫色能够自然且模糊地渗透进面板背景中，卡片色自动跟随环境色温进行“温润呼吸”，大大提升了高级感。

### 1.3 PC 端侧边栏吸附 100% 满铺填充 (Zero Margin Docking)
* **无缝贴合**：控制栏在拖拽吸附到侧边栏（TR/BR/TL/BL）后，取消卡片悬浮样式。强制设置 `bottom: 0; right: 0; border-radius: 0; box-shadow: none; border: none;` 并将宽度拓宽至 `var(--sidebar-width)`。
* **净高对齐计算**：同步修改 `DragManager.js` 的排版占位计算，停靠后将缩进变量 `--docked-controls-height` 由原本的 `h + 32px`（含对冲边距）修正为纯净的面板高度 `h`，使得评论区列表被推起后与控制板完美贴合，周围不再留有任何无意义的物理缝隙。

### 1.4 跳转按钮防挤压布局优化
* **侧边栏宽度限制**：为了保障评论区与控制面板的功能性体验，将 PC 横屏侧边栏的响应式 clamp 最小宽度值由 `320px` 调整为 `350px`。
* **锁定两行防换行**：控制面板在 PC 模式下的 `min-width` 升级为 `348px`，通过 CSS 强行锁死卡片底层尺寸，完美确保快进/快退/快退组在极端窄屏侧边栏环境下不会因宽度被过度压榨而将 6 个按钮换行挤压成 3 行。

### 1.5 活跃标签页胶囊内进度条显示
* **行内渐变进度映射**：当区间标签被激活并执行循环播放时，在 `timeupdate` 的循环中根据 `(currentTime - startTime) / duration` 计算实时百分比 `pct`。
* **零 DOM 开销**：直接通过 JS 改变该活跃标签的 style 属性 `background`，使用无渐变过渡的双色线性渐变 `linear-gradient(to right, hsla(var(--tab-color), 0.38) pct%, hsla(var(--tab-color), 0.15) pct%)` 动态渲染进度，并在停靠/取消循环时立即通过 `_clearAllTabProgress()` 清除，效率极高。

### 1.6 交互逻辑修复
* **点击取消循环时不闪烁开始位置**：将 `showJumpHint` 的触发逻辑从 tab 点击总入口中移出，分别放入 Highlight 标签跳转和区间 Loop 标签激活两个实际发生 timeline 寻址（Seek）的分支中。当再次点击当前活跃胶囊以“取消循环”时，视频保持原地播放，进度条不再闪烁无意义的开始位置提示。

### 1.7 URL 时间戳/参数自动清理防冲突
* **静默地址净化**：当脚本解析完 URL Hash/Search 参数并成功导入为本地循环标签后，立即自动剔除 URL 尾部的 `#时间戳`（或 `#t=...`）以及 `?t=...`、`?start=...`、`?end=...` 等时间查询参数。
* **阻止宿主自动循环**：通过在不刷新页面的情况下静默重写浏览器地址栏，彻底切断了 MissAV 宿主网站原生脚本检测 Hash 变化以自动触发原生循环或播放跳转的逻辑通道，避免了第三方逻辑与我们播放器脚本之间的时序冲突。

## 2. 踩坑教训与开发经验

### 2.1 剔除无意义的 `timeupdate` 性能开销
* **现象**：原先的系统使用 `_updatePassiveTabGlow()`，在视频播放的 `timeupdate` 事件下对所有非编辑态的标签胶囊进行 DOM 查询，并高频执行接近度计算（±5秒内亮起），这带来了潜在的卡顿。
* **优化**：在去除该“无聊发光特效”后，果断注销了该时间监听。新设计的胶囊内进度条只通过事件驱动对**有且仅有的一张活跃标签**进行 inline style 的精细化更新，大幅降低了主线程开销。

### 2.2 解除 Compositing Layer 剪裁限制的伪元素冗余
* **现象**：早期为了防止圆角加高斯模糊引起浏览器合成层渲染时剪裁掉内部标签的 `box-shadow`，引入了复杂的 `::before` 视觉剥离方案。
* **优化**：根据“Ponytail”规范，既然我们已经将标签胶囊的所有高光、呼吸特效和繁重阴影全部剔除为干净的扁平实色，子元素不再有任何溢出投影。因此不再需要为父卡片配置复杂的伪元素背景分离。将属性回归原生类名不仅极大地缩短了 CSS 链长，也让后续维护变得非常简单直观。

### 2.3 `min-width` 与 `max-width` 声明冲突
* **经验**：在 PC 侧边栏中，原本使用了 `width: calc(var(--sidebar-width) - 32px)`，但为了防止窄屏挤压按钮，我们引入了 `min-width: 348px`。在 CSS 规范中，**当 `min-width` 声明的值大于 `max-width` 或 `width` 时，`min-width` 会无条件享有最高优先级**，这免去了我们用繁琐的 `@media` 去额外覆写 `max-width` 的烦恼，使得锁死最小宽度的逻辑写起来更精简。

### 2.4 使用 `replaceState` 静默更新 URL
* **经验**：对于需要清理 URL 锚点和查询参数，同时绝对不能刷新页面（重新加载视频与脚本）的需求，`window.history.replaceState` 是唯一的标准解。通过利用 `URL` 和 `URLSearchParams` 类删除对应的键，再进行替换，保证了地址栏状态的精确和宿主页面逻辑的平静。
