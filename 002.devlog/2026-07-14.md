# 开发日记 — 2026年7月14日
## 多标签高光标记与区间循环（Tab-Style AB Loop）系统重构

---

## 1. 核心需求与实现功能

### 1.1 标签页视觉与交互架构重构
- **紧凑横向标签栏**：将原先占位置的静态时间轴控制栏，重构为可横向滑动的浏览器标签页式设计（`.tm-tab-scroll-container`），右侧固定悬浮 `[ + ]` 新增按钮。
- **排他性编辑与对齐**：当进入编辑模式（长按）或新建草稿模式时，自动开启“独占渲染”，隐藏其余无关标签。由于此时编辑胶囊是容器内的唯一元素，它会自然紧贴容器最左侧对齐，为 A、B 打点插槽和备注提供了最宽裕的物理宽度，完美防止布局拥挤。
- **按时间年代排序**：标签页在渲染前会根据 `startTime` 进行严格的从左到右升序排列。

### 1.2 极简编辑/草稿胶囊设计
- **体积微型化**：通过紧凑的 padding（`3px 8px`）、小字号（`11px`）以及暗黑透明圆角背景块（`rgba(0, 0, 0, 0.25)`）分别封装 A 和 B 框，大大压缩空间。
- **无边框纯图标按钮**：取消繁琐的背景阴影与实体按键框，修改备注 `✏️`、保存 `✔` 使用纯文字图标；取消 `✕` 采用微红色半透明背景，精致且极简。
- **时间录入控制**：点击 `+` 按钮时，立即将当前播放位置填入 A 栏，且 B 栏在 A 栏未设点时处于禁用半透明状态。限制 B 必须严格大于 A。
- **备注 Cancel 逻辑**：重写了修改备注的弹框行为，将原先清空备注的 "Skip" 按钮改名为 "Cancel"，点击取消不修改原先的备注数据。

### 1.3 协调一致的动态色彩系统
- 设计了一套包含 **6 种柔和高亮 HSL 主题色** 的调色板：
  - 蓝色 (`200 100% 55%`)、绿色 (`145 80% 48%`)、紫色 (`260 85% 62%`)、珊瑚色 (`15 95% 58%`)、粉色 (`330 90% 60%`)、薄荷绿 (`170 85% 42%`)。
  - 每个标签页按顺序轮流继承这些颜色，并动态应用到对应的进度条高光上。

### 1.4 进度条标记系统（完全按草图设计）
- **单时间戳 (Highlight)**：渲染在进度条内部的微型垂直指示柱（`width: 6px; height: 3px; bottom: -4px;`），高度和垂直基准线完全与区间样式对齐。
- **未激活区间 (Inactive Range)**：渲染在进度条底部边缘下方的极细线条（`height: 3px; bottom: -4px;`），平时为 `0.25` 低透明度，悬浮时变为 `0.45`。
- **激活区间 (Active Looping Range)**：激活后图层拉伸到与进度条同高，并简化为**双图层重叠显示**：
  - **下层（区间背景）**：覆盖整个区间（`A ~ B`），显示为 `0.15` 透明度的浅色底。
  - **上层（区间已播进度）**：覆盖从 `A` 点到当前播放进度 `currentTime`，显示为 `0.45` 透明度的色条。
  - **边界**：两端分别添加两个 `4px` 宽度的边界指示柱以示区分。

### 1.5 旧版本 URL 的向下兼容与本地存储转换
- 重写 URL 参数转换器，兼容各种旧版链接，自动转换并保存为本地 LocalStorage 标签：
  - **支持多参数渠道**：同时监听 `hash` 变化与 `search` 查询字符串（如 `?t=...` 或 `?start=...&end=...`）。
  - **支持多格式解析**：`hh:mm:ss`（如 `00:01:15`）、`mm:ss`（如 `01:15`）以及纯秒数数字（如 `75`），并防重落库。

## 2. 踩坑教训与开发经验

### 2.1 CSS 变量 HSL 逗号与斜线语法混用冲突 (重大教训)
- **现象**：图层和标签页背景颜色在设置了透明度 `hsla(...)` 后莫名变为完全透明（不可见）。
- **原因**：在 JS 中定义的色值包含逗号（如 `200, 100%, 55%`），当其代入 `hsla(var(--tab-color) / 0.25)` 时，渲染结果为 `hsla(200, 100%, 55% / 0.25)`。在现代 CSS 规范中，**如果在 `hsla()` 中使用了传统的逗号，就绝对不能再使用斜线 `/` 作为透明度分隔符**，否则浏览器会将其判定为无效语法直接丢弃。
- **解决方法**：依照 CSS Color Level 4 规范，将 JS 颜色定义全部修改为无逗号的空格分隔语法（如 `200 100% 55%`）。这样 `hsla(200 100% 55% / 0.25)` 能够被现代浏览器完美解析。

### 2.2 SPA 架构中视频元数据加载时序 fallback
- **现象**：当播放器刚刚初始化或 SPA 页面切换视频时，进度条上的标记无法渲染。
- **原因**：初始状态下，视频时长 `duration` 为 `NaN` 或 `0`，代码直接 return 拦截。同时由于单页应用环境切换时，原本的 `loadedmetadata` 事件可能早已提前触发完毕，导致无法重复捕捉。
- **解决方法**：实现了一套 `timeupdate` 的一次性降级监听（Fallback）。一旦检测到时长缺失，就监听 `timeupdate` 并在时长第一次出现有效数字的瞬间渲染标记，随后自我注销，极大地增强了在宿主网站下的容错率。

### 2.3 鼠标事件阻挡与 Seek 拖动体验 (交互经验)
- **现象**：进度条添加了图层容器 `.tm-progress-markers-container` 置顶显示（`z-index: 5`）后，用户无法直接点击进度条进行视频快进/后退（Seek）。
- **经验**：对于覆盖在原生控制轴之上的图层容器，必须使用 **`pointer-events: none;`** 进行穿透。
  - 只对有实质跳转交互的静态标记（如 `.tm-progress-marker-tick` 和 `.tm-progress-marker-range`）显式设置 `pointer-events: auto;`。
  - 对于处于激活循环状态的区间图层，为了不干扰用户在区间内拖拽或点击微调进度，其 `pointer-events` 也应设为 `none`，从而实现了极佳的免打扰 Seek 拖曳手感。

### 2.4 模块循环依赖解耦 (架构经验)
- **经验**：原本的 `LoopManager` 和大型 `CommentScraper` 之间存在紧密的相互引用（循环依赖）。通过将核心的时间字符串解析和 AV 号转换函数提取到独立的工具类 `utils/videoCode.js` 中并单独暴露，彻底斩断了循环引用链条，为 Webpack 的模块化打包扫清了障碍。
