* **Webpack GM 跨域白名单扩展**：

* **异步并发请求 (`Promise.allSettled`)**：
  * 评论面板初始化与刷新时，采用 `Promise.allSettled([loadJableComments(1), loadJavlibComments(1), loadJavdbComments(1)])` 并行异步发起 Jable.tv、JAVLibrary 与 JavDB 三路评论采集，互不阻塞。
* **智能 Section 动态排序 (Dynamic Section Sorting)**：
  * 在 `renderCommentsList()` 中实现稳定排序算法：默认板块顺序为 `[Jable.tv, JavDB, JAVLibrary]`。
  * 当某平台无评论（0 条）、未收录（404）或加载不可达时，该 Section 自动下沉移至列表末尾，确保有有效评论的平台优先呈现在用户最可视区域。
* **自动联动展开 (Auto Expansion)**：
  * 首次加载完成后，若默认排第 1 位的 Jable 无评论，系统将自动识别并展开首个拥有评论的后置板块（如 JavDB 或 JAVLibrary）。

## 2. 踩坑教训与避坑经验

### 2.1 第三方 API 签名 MD5 校验的严格规范 (RFC 1321)
* **坑点**：此前手写或简化的 MD5 算法在处理纯 ASCII 较短字符时正常，但是在拼接了长字符串 Salt 或 UTF-8 特殊符号后，补位计算逻辑（padding bits & length bits）与标准算法出现偏差，导致计算出的哈希摘要与服务端不一致，抛出 `InvalidSignature` 错误。
* **经验**：涉及 HMAC、签名校验或加密算法时，切勿凭感觉手载简略算法，务必使用经过 RFC 标准测试向量验证的完整算法实现（如 Joseph Myers RFC 1321 规范算法），避免因边界条件差异陷入排查困境。

### 2.2 暗黑模式 (Dark Mode) 屏显文字“发光发亮感”与视觉疲劳成因
* **坑点**：纯黑背景（`#000000` / `#0A0A0C`）下直接使用 100% 纯白文本（`#FFFFFF`）会导致人眼视网膜产生光晕扩散现象（Halation Effect），使得文字显得粗笨发胀，长时间阅读极易疲劳。
* **经验**：在暗色模式下，主正文文本透明度应控制在 `85%~90%`（如 `rgba(255,255,255,0.88)` / `#E1E1E1`）；辅助文字（如作者、时间）透明度降低至 `40%~65%`；配合 `1.5~1.6` 的舒适行高与 `0.01em` 微字距，可瞬间提升 UI 的高级感与易读性。

### 2.3 多源异步数据流与 DOM 伸缩渲染的解耦
* **坑点**：在异步并发请求多源数据时，若每个源返回后直接销毁并全量重新创建 DOM 结构，会导致正在滚动的其他源板块 ScrollTop 重置或产生严重的闪烁。
* **经验**：在重新渲染组合式多 Section 列表前，应先记录每个 `.tm-comment-section-body` 的当前 `scrollTop`，在 DOM 节点更新后统一还原滚动位置。同时，对展示顺序采用稳定排序规则（Stable Sort），避免因异步响应返回顺序不同而发生视觉上的“板块位置跳变”。

### 2.4 面向对象重构中的渐进式抽象与 Getter/Setter 桥梁模式
* **坑点**：大型类（如 `CommentPanel.js` 3000+ 行）如果直接重命名所有外部/内部引用的变量，极易漏改某些角落导致运行时 `undefined` 报错。
* **经验**：在将散乱变量重构为统一字典对象（如 `this.sites`）时，可以在类构造函数中使用 `Object.defineProperty` 为原有变量挂载 Getter/Setter 桥梁。这样既享受了底层抽象集约化的好处，又完全避免了上层重写引发 Regression BUG 的风险。
