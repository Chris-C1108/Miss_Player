## 1. 核心需求与实现功能

### 1.1 评论区底部悬浮发表组件 (`.tm-comment-submit-bar-wrapper`) 与标签选择模态框
* **悬浮发表栏 UI 重构**：
  * 引入全新的底部发表行组件 `.tm-comment-submit-bar-wrapper`，内含精确居中的矢量 `+` 按钮、单行文本输入框以及基于全局 i18n 系统的 `${__('send')}` 发送按钮。
  * 彻底移除了原有的红框边框与强红色调，改为现代暗色半透明风格，背景采用向上柔和渐变过渡 (`linear-gradient(to top, rgba(10, 10, 12, 0.98)...)`).
* **即时标签勾选弹出框 (`.tm-comment-tag-select-modal`)**：
  * 点击 `+` 按钮实现 **0ms 零延迟** 弹起标签勾选模态框（含顶栏【全选 / 取消全选】），支持直接勾选及修改高光点/区间标签备注。
  * 采用底部向上位移并淡入的平滑动画（`translateY(16px)` $\to$ `translateY(0)`）。
* **标准评论格式输出与分隔符统一**：
  * 点击发送时自动将选中的标签与备注拼接为标准评论格式：
    ```text
    [00:00:42] 备注xxx ；
    [00:02:42 ~ 00:07:42] 备注xxx ；
    评论内容
    ```
  * 统一时间区间的格式分隔符为 **`~`**（例：`[00:02:42 ~ 00:07:42]`）。
* **实时权限与登录状态校验 (`checkCanComment`)**：
  * 在点击发表或弹窗按钮时，实时校验目标站点、视频收录情况与登录状态；未登录时弹出登录提示。

### 1.6 评论区 Header 站点登录状态指示器 (`.tm-comment-login-badge`) 与零延迟 SWR 架构
* **站点 Header 登录指示器**：
  * 在评论面板各分区 Header（如 Jable.tv）左侧 `tm-comment-status-badge` 旁，自动渲染当前站点的登录状态 Badge。
  * 对于不支持登录的站点（如 JAVLibrary），智能判断并隐匿该指示器。
  * 状态分为：
    * **已登录** (`.is-logged-in`)：绿色微光底色，提示正常登录；
    * **未登录** (`.is-not-logged-in`)：橙蓝色提示，鼠标悬停上浮高亮。点击即阻断 Header 折叠事件 (`e.stopPropagation()`) 并弹窗该站点的专属登录模态框；
    * **检测中...** (`.is-checking`)：灰色淡化风格，用于无任何历史缓存时的首帧平滑过渡。
* **0ms 零延迟 SWR (Stale-While-Revalidate) 架构**：
  * 引入 `cacheLoginStatus` 与 `getCachedLoginStatus` 持久化读写机制（结合 Tampermonkey `GM_setValue` / `localStorage` 以及凭据有效性预测）。
  * 评论面板在首帧（0ms）同步读取缓存即刻渲染最终 UI，杜绝原本因跨域 HTML GET 网络请求（1~3秒）导致的初始显示“未登录”而后突变“已登录”的延迟卡顿与 UI 闪烁。
  * 在 `CommentPanel` 中接入 `JableLoginProvider` 与 `MissavLoginProvider` 静态 Provider 兜底，解耦 `window.loginManager` 的 async 初始化，确保首帧 100% 成功读取状态。
* **跨域 Cloudflare 校验误杀防护**：
  * 修复在第三方站点（如 MissAV）跨域探测 Jable 登录态时，因遇到 Cloudflare 人机验证屏障（CF Challenge / 403）导致误判定为“未登录”并覆盖缓存的问题。
  * 在 `JableLoginProvider.js` 中增加 CF 屏障防护，遇到验证拦截时自动保持历史缓存状态。
* **网络并发解耦与 60s 频控锁 (Throttling)**：
  * 将登录态复核请求从 `renderCommentsList()` 的频繁触发中完全剥离，并加入 60 秒频控锁 (`_lastStatusCheckTime`)。
  * 首次打开面板时延迟 1.5 秒发起后台复核，避免与评论数据抓取请求争抢网络连接池带宽。
* **退出播放器宿主 `site-header` 残留隐藏 BUG 修复**：
  * 修复从播放器右上角关闭按钮或滑动退出后，宿主站 Jable.tv 的 `.site-header` 依然消失不显示的问题。
  * 补全 `PlayerCore.js` `close()` 方法中对 `tm-player-active` / `controls-hidden` 状态类名的清理，恢复宿主 Header CSS 样式并补发 `scroll` / `resize` 事件通知宿主重绘。

## 2. 踩坑教训与避坑经验

### 2.1 彻底抛弃绝对定位悬浮遮挡方案，重构为 Flex In-Flow 流式布局
* **坑点**：此前将发表栏和次要功能栏设为 `position: absolute; bottom: 0;` 覆盖在评论列表上方。在单分区下虽然可通过 padding 规避，但在多分区（Jable + JAVLibrary + JavDB）且各分区高度动态伸缩时，位于底部的分区（如 JAVLibrary）的 Header 与末尾评论必然会被遮挡。
* **经验**：对于多卡片/多分区的列表容器，底部的固定操作栏应采用 Flex 容器的内联流式子项 (`flex: 0 0 auto; position: relative`)，将滚动责任全权交给 Flex item (`flex: 1; overflow-y: auto`)。这不仅天然保证了滚动区域绝不重叠，还能避免各种计算复杂的动态 bottom offset。

### 2.2 异步网络检测与同步 UI 弹窗响应解耦
* **坑点**：最初在点击 `+` 按钮弹起标签选择框时，由于调用了 `checkCanComment()`（内含网络检测与登录判断 Promises），导致用户点击 `+` 按钮后出现明显的 1 秒卡顿与延迟感。
* **经验**：UI 交互响应（如弹窗/展开）必须维持同步 0ms 级的即时反馈，网络状态或权限校验应当异步进行或在最终点击【发送/确认】时进行二次拦截，切勿让 async 网络开销阻塞 UI 层的交互流畅度。

### 2.3 伸缩容器中文本变长导致邻近按钮折行变形 (Flex Squeezing Issue)
* **坑点**：次要功能栏在动态插入 `正在采集 JAVLibrary...` 长文本时，左侧容器宽度膨胀，强行挤压了右侧的 `[复制全部]` 按钮，导致文字被挤成两行（`复制` \n `全部`）。
* **经验**：在包含动态文字和固定按钮的单行工具栏中，文本节点必须设置 `min-width: 0; flex: 1; overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`，而按钮节点必须设置 `flex-shrink: 0; white-space: nowrap;`，这是保证单行工具栏防变形的黄金 CSS 组合拳。

### 2.4 CSS `:has()` 伪类在 DOM 兄弟节点跨层通信中的妙用
* **坑点**：`.tm-handle-container` 是在 DOM 树中位于 `.tm-comments-panel` 之前的兄弟节点，普通 CSS 无法向后选择（无法用 `+` 或 `~`）。在以前需要写大量 JS 在控制面板显示/隐藏时同步操作手柄节点。
* **经验**：利用现代 CSS 的 `:has()` 选择器（如 `.tm-player-container:has(.tm-comments-panel.is-dimmed) .tm-handle-container::after`），可以直接在父容器监听到后代子节点的类名变化并反向样式化其他子节点，既减少了 JS 依赖，又达到了零闪烁的完美渲染效果。

### 2.5 跨域 DOM/HTML 探测切勿把“验证拦截”或“网络失败”等同于“未登录”
* **坑点**：在 MissAV 跨域检查 Jable 登录态时，由于 `GM_xmlhttpRequest` 触发了 Jable 的 Cloudflare 屏障，返回了带有 `cf-challenge` 的 403 页面。原本逻辑未区分“普通未登录页面”与“异常屏障页面”，直接执行 `cacheLoginStatus(false)`，导致用户的“已登录”状态被误杀覆盖，产生“已登录”闪烁变为“未登录”的 BUG。
* **经验**：跨域探测登录态时，必须先对响应 HTML 进行防线排查（检查是否包含 `cf-challenge` / `Cloudflare` / `Just a moment` / `403` / `503`）。遇到拦截或请求失败时，应“保持现有缓存（Stale State）”而非推翻重置，避免异常覆盖真实的登录记录。

### 2.6 频繁 UI 重绘逻辑中切勿包含无频控的异步网络请求
* **坑点**：此前在 `renderCommentsList()` 函数中直接调用了 `updateAllSiteLoginStatuses()`。由于切换页码、过滤灌水、评论抓取进度更新等操作均会触发 `renderCommentsList()`，导致瞬间向目标服务器打出大量重复的跨域 GET 请求，不仅造成严重的连接池拥堵、拖慢评论抓取速度，更触发了服务器的频繁限速与拦截。
* **经验**：网络请求（特别是跨域探测）必须与 UI 渲染生命周期严格分离。UI 重绘应仅从本地内存/缓存中同步读取变量，网络复核需通过独立函数触发，并必须配备**频控锁（如 60s 冷却）**与**延时错峰（如 1.5s 避开数据抓取高峰）**。

### 2.7 模块化架构中全局状态类名清理必须下沉至 Core/Base 层
* **坑点**：先前移除 `tm-player-active` 的代码仅写在 `CustomVideoPlayer.prototype.close()` 中，而点击界面右上角 `×` 按钮时，事件直接响应调用的是 `PlayerCore.prototype.close()`。这导致从 GUI 按钮退出时，`tm-player-active` 类名没有被移除，CSS 规则 `body.tm-player-active header { display: none !important; }` 持续生效，使得 Jable 宿主站的 `site-header` 在退出播放器后彻底隐藏消失。
* **经验**：在复杂项目中，所有影响宿主 DOM/CSS 全局环境的操作（如 `document.body` 类的增删、主题色修改、隐藏滚动条 Style 节点的注入等），其还原与清理逻辑必须收拢并下沉到最底层的核心类 (`PlayerCore.close()`) 中，确保无论从哪个入口（快捷键、手势滑动、界面按钮、系统销毁）退出，都能保证 100% 的幂等清理。

### 2.8 异步模块未就绪时的 UI 依赖与静态实例兜底 (Static Fallback Pattern)
* **坑点**：`window.loginManager` 属于异步初始化的全局模块。评论面板在刚挂载并开始抓取数据（显示“采集中...”）的第 1 帧，由于 `window.loginManager` 仍在 `initAutoLogin()` 的 Promise 中，`getLoginProviderForSite` 查不到 Provider 实例返回 `null`，导致面板误以为不支持登录而不渲染指示器。直到几秒后评论抓取完毕再次触发渲染时，恰好 `loginManager` 已就绪，才突兀地渲染出 Indicator，给用户造成“非要等评论采集完才显示登录状态”的错觉。
* **经验**：组件在查询外部异步管理器时，应提供本地静态/单例 Provider 兜底（Static Fallback Pattern）。即便全局 Async Manager 尚未就绪，也能凭借静态 Provider 同步读取到 LocalStorage 里的持久化缓存并完成首帧渲染，彻底消除依赖链上的时序死锁与错觉延迟。
